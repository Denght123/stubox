# 高校招生 AI 问答系统 PRD

## 1. 项目背景

本项目面向河北水利电力学院招生咨询场景，提供免登录、轻量、极速的 AI 问答入口。系统重点解决高三毕业生和家长在移动端与 PC 端快速咨询招生政策、专业介绍、校园生活等问题的需求。

## 2. 产品目标

- 打开即用：不注册、不登录、不创建账号。
- 响应迅速：后端采用 OpenAI 兼容流式调用，前端逐字渲染。
- 高可用：主模型失败时自动切换备用模型。
- 内容可控：通过根目录 `agent.md` 统一约束招生助手身份和回答边界。
- 视觉现代：参考 Gemini 浅色、蓝白渐变、极简聊天界面，融合学校校徽但不过度抢眼。

## 3. 用户与场景

- 高三学生：咨询专业、录取规则、校园生活、住宿条件。
- 家长：咨询学校办学特色、就业方向、奖助政策、报到流程。
- 招生工作人员：通过修改后端配置和 `agent.md` 快速调整模型与回答口径。

## 4. 功能范围

### 4.1 必做功能

- 单会话聊天：仅保留当前页面的一组问答。
- 前端 Session 保存：刷新页面后短期保留本次会话，不落数据库。
- SSE 流式输出：AI 回复逐字显示。
- OpenAI 兼容配置：后端预留 `BASE_URL`、`API_KEY`、`PRIMARY_MODEL`、`BACKUP_MODEL`。
- 主备模型容灾：主模型请求失败后自动重试备用模型。
- 热读系统提示词：每次请求读取根目录 `agent.md`。
- 移动端适配：底部输入托盘、触控按钮不小于 44px。

### 4.2 不做功能

- 用户登录、注册、权限体系。
- 数据库存储、多会话管理、云端历史同步。
- 用户自定义模型供应商或 API Key。
- 管理后台、复杂知识库上传、账号计费。

## 5. 体验要求

- 首屏是实际聊天界面，不做营销落地页。
- PC 端保持左侧轻量品牌栏和主聊天区。
- 移动端优先展示品牌、欢迎语、快捷问题和底部输入框。
- 学校 logo 与校名只作为品牌信号，保持低干扰。
- 错误和备用模型切换要给出轻提示，不打断用户输入。

## 6. 成功标准

- 本地启动后可访问前端页面并发送消息。
- 后端在未配置 API Key 时给出可读错误，不崩溃。
- 配置有效模型后能够流式返回内容。
- 主模型 4xx、5xx、超时或网络异常时会自动尝试备用模型。
- 前端在 375px 手机宽度和桌面宽度下无横向滚动、无遮挡。
